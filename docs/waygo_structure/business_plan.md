# Waygo Business Plan

## 1. Executive Summary

**Waygo** is a gamified city exploration mobile app for Plovdiv, Bulgaria. The app transforms exploration into an adventure game where users discover local businesses, complete themed quests, earn XP and vouchers, and compete on leaderboards.

**Mission:** "Turn every city exploration into an adventure"

**Investment Ask:** Bootstrap initially, break-even by Month 8, seeking 25,000 BGN Year 1 revenue

---

## 2. Problem

- **Tourist concentration:** 70% visit only top 5 attractions; local businesses miss foot traffic
- **Discovery friction:** No engaging way to find authentic local experiences
- **Business invisibility:** Local cafes, museums, restaurants invisible against Google Maps chains
- **Shallow tourism:** Visitors miss authentic Plovdiv; economy flows to chains not locals

---

## 3. Solution

- **Fog-of-War Map:** Hidden areas revealed only when explored (gamified discovery)
- **Quest System:** Themed challenges guiding users to multiple businesses (Coffee Trail, History Hunter, Grand Tour)
- **Check-in Validation:** GPS + code verification for authentic visits
- **Gamification:** XP, levels, streaks, badges incentivizing return visits
- **Voucher Marketplace:** Real discounts from partner businesses
- **Leaderboards:** Weekly/monthly/all-time social competition

---

## 5. Product Description

**Core Features:**
1. Interactive dark-themed map with fog-of-war overlay
2. Check-in system (GPS + manual code entry)
3. Quest builder with waypoints and XP rewards
4. Gamification engine: XP, levels, streaks, badges
5. Partner directory with categories (cafe, museum, cultural, restaurant)
6. Leaderboard with weekly/monthly/all-time views
7. Voucher system for earned discounts
8. Admin and partner dashboards

**Technology:** React 18, Leaflet/OpenStreetMap, Tailwind CSS (currently prototype)

## 14. Strategic Recommendations

1. **Priority:** Secure 20 committed partners before launch (validation critical)
2. **Partnership:** Approach Plovdiv Tourism Board early (official endorsement)
3. **Focus:** Prioritize quest quality over quantity (user retention depends)
4. **Expansion:** Validate in Plovdiv before scaling (prove model first)
5. **Revenue:** Quest sponsorship easiest early revenue stream (before subscriptions)

---

## Weaknesses & Improvements

### 3 Major Weaknesses Identified:

1. **Dependent on partner supply for value** - Need critical mass before user value materializes
2. **Heavy reliance on repeat users** - Streaks drive engagement but create churn risk
3. **Check-in integrity** - GPS/code can be gamed or faked

### Suggested Improvements:

- Add photo verification at check-in (require location + timestamp photo)
- Partner referral bonuses (free month for each new partner)
- Social sharing incentives (bonus XP for sharing streaks)
